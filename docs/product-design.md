## 1. 产品定位

### 1.1 产品概述

内容生产流水线系统（Content Pipeline System）是一套面向 AI 技术类抖音账号的**半自动化内容生产平台**。它将一条视频从选题到发布的 7 个阶段标准化、模块化、可编排，通过 Java 平台 + Python AI 服务的混合架构，实现"选题采集 → 脚本生成 → 分镜拆解 → 视频生成 → 配音 → 剪辑 → 发布"的全流程管理。

### 1.2 核心价值

| 维度 | 说明 |
|------|------|
| **提效** | 一条视频从 2-3 小时缩短到 30 分钟以内（含人工审核） |
| **标准化** | 7 个阶段全部标准化，新人培训成本大幅降低 |
| **可扩展** | 支持多账号、多平台、多模型，矩阵运营 |
| **可追溯** | 每条视频的全流程状态可见，问题环节一目了然 |

### 1.3 目标用户

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| **内容运营** | 负责选题、审核成片 | 热点选题推荐、快速审核、一键发布 |
| **视频编辑** | 负责分镜、剪辑 | 分镜模板、素材管理、批量剪辑 |
| **管理员** | 负责系统配置、账号管理 | 模型配置、平台接入、权限管理、数据统计 |
| **AI 系统** | 后台自动化服务 | 脚本生成、视频生成、配音、字幕等自动化任务 |

## 3. 业务流程

### 3.1 核心流程 — 一条视频的生命周期

```
[选题] → [脚本] → [分镜] → [素材] → [配音] → [剪辑] → [发布]
  │         │         │         │        │         │        │
  ▼         ▼         ▼         ▼        ▼         ▼        ▼
 状态机流转 (task_status)
```

### 3.2 详细状态流转

```
                  ┌─────────────┐
                  │   WAIT      │  ← 创建任务，等待处理
                  └──────┬──────┘
                         ▼
                  ┌─────────────┐
           ┌─────│  SCRIPTING   │  ← AI 生成脚本
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  SCRIPT_REVIEW │  ← 人工审核脚本
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  STORYBOARD │  ← 拆解分镜
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  GENERATING │  ← 生成视频/图片素材
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  VOICEOVER  │  ← AI 配音
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  EDITING    │  ← FFmpeg 剪辑合成
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  REVIEW     │  ← 人工终审（30 秒）
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  READY      │  ← 待发布
           │     └──────┬──────┘
           │            ▼
           │     ┌─────────────┐
           │     │  PUBLISHED  │  ← 已发布
           │     └─────────────┘
           │
           └───── ERROR ← 任意步骤失败
```

### 3.3 人工介入点

| 阶段 | 人工介入 | 预计耗时 |
|------|----------|----------|
| 选题 | 确认选题价值 | 2 分钟 |
| 脚本 | 审核 AI 生成的脚本 | 3 分钟 |
| 分镜 | 微调分镜描述 | 5 分钟 |
| 素材审核 | 挑选满意的生成结果 | 5 分钟 |
| 终审 | 审核成片是否符合预期 | 30 秒 |

## 5. 产品路线图

### Phase 1 — MVP（4-6 周）

**目标：跑通核心流程，人工为主 AI 为辅**

- 选题手动录入
- 脚本 AI 生成 + 人工审核
- 手动分镜 + AI Prompt 生成
- 接入一个视频生成模型（首选可灵）
- AI 配音（豆包）
- FFmpeg 剪辑合成 + 自动字幕
- 人工审核发布
- 用户权限管理

### Phase 2 — 自动化（2-3 周）

**目标：减少人工介入，提升生产效率**

- 热点自动采集（GitHub Trending / Hacker News / RSS）
- 自动拆解分镜
- 多模型并行生成（可灵 + Veo + 即梦）
- 定时发布
- 素材库管理

### Phase 3 — 矩阵运营（2-3 周）

**目标：支持多账号多平台矩阵运营**

- 多平台账号管理
- 矩阵发布策略
- 评论区管理
- 数据分析
- 运营日报

### Phase 4 — 智能化（持续迭代）

**目标：数据驱动选题与优化**

- 热点智能推荐（基于历史数据）
- 视频效果预测
- A/B 测试封面/标题
- 自动优化 Prompt 模板
- 语音克隆

## 6. 成功指标

| 指标 | 目标值 |
|------|--------|
| 单条视频生产耗时 | < 30 分钟（含人工审核） |
| 人工审核占比 | < 20% 总耗时 |
| 日均产量 | ≥ 10 条/账号（成熟期） |
| 爆款率（点赞 > 1000） | ≥ 5% |
| 系统可用性 | ≥ 99.5% |
